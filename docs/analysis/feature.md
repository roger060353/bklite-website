# 功能介绍

## 1. 分析视图（View）

分析视图是运营分析的主工作台，采用**左侧目录树 + 右侧内容区**的双栏布局，负责承载目录管理、分析对象切换和具体内容展示。相比单页报表工具，BlueKing Lite 运营分析更强调「资产化管理」，让团队能够围绕业务场景持续沉淀可复用的分析资产。

### 1.1 目录与分析资产管理

#### 核心定位

目录是分析资产的组织载体，决定了团队如何沉淀内容、如何共享页面，以及如何把临时分析转化为长期可复用资产。平台通过树形目录统一管理**目录、仪表盘、拓扑图、架构图**四类对象，支持多层级结构，便于按业务域、专题或职责范围进行组织。

#### 核心能力

*   **统一树形组织**：在同一目录树中统一管理四类分析对象，避免内容散落在多个孤立页面
*   **三级层级约束**：目录层级上限为 3 层，在灵活性与可维护性之间取得平衡
*   **同父目录唯一性**：同一父目录下目录名称唯一，防止命名冲突导致的检索困难
*   **就地创建与维护**：支持在目录树中直接新增、编辑、删除目录和分析对象，减少页面切换成本
*   **删除保护机制**：目录下存在子对象时不可直接删除，避免误操作破坏已沉淀的分析结构
*   **未保存变更提醒**：切换分析对象或离开页面时，系统对未保存变更进行二次确认，防止编辑成果丢失
*   **启用/停用控制**：目录支持状态管理，停用后不在目录树中展示，便于软删除与归档

>

>
> *   **图表解读**：目录树不是简单导航栏，而是分析资产的管理入口。通过目录层级与权限配置，实现分析成果的团队级沉淀与可控共享。

### 1.2 仪表盘

#### 核心定位

仪表盘是面向**指标观察**的核心载体，适合承载资源总览、趋势对比、专题观察等场景。基于网格布局自由组合多种可视化组件，是运营分析中最适合做日常巡检和运营总览的展示形态。

#### 核心能力

*   **网格化自由布局**：支持拖拽调整组件位置与大小，灵活组织页面结构
*   **多图表类型支持**：
    *   **折线图**：适合趋势观察、时序对比
    *   **柱状图**：适合分类对比、排名展示
    *   **饼图**：适合占比分析、构成展示
    *   **单值图**：适合关键指标、核心状态展示
*   **全局时间驱动**：当页面包含时间型参数时，可通过顶部全局时间选择器统一驱动多个组件联动刷新，便于跨指标横向对比
*   **数据源模板复用**：组件配置时可复用数据源中定义的参数模板，自动识别固定值、可编辑值等参数类型，降低重复配置成本
*   **公共过滤条件**：支持配置页面级公共过滤器，统一作用于页面内多个组件
*   **保存与手动刷新**：支持页面配置持久化保存，支持手动触发刷新，适合值班巡检和专题回看场景

> *   **图表解读**：仪表盘的价值在于将零散指标组织成可读的运营视图，通过时间轴对齐和布局编排，帮助用户快速建立对系统状态的宏观认知。

### 1.3 拓扑图

#### 核心定位

拓扑图是面向**关系定位**的核心载体，用于表达对象之间的连接关系、依赖链路和状态分布。当问题不仅体现在数值变化，还需要理解「问题在哪里、与谁相关、影响范围多大」时，拓扑图比纯数字看板更能帮助处理人建立问题上下文。

#### 核心能力

*   **五类节点灵活编排**：
    *   **图标节点**：使用预设图标表达资源类型
    *   **基础图形**：矩形、圆形等几何图形表达抽象节点
    *   **文本节点**：纯文本说明与标注
    *   **单值节点**：绑定数据源展示关键指标数值，支持阈值配色
    *   **图表节点**：在节点内嵌迷你图表，展示趋势变化
*   **连线与标签表达**：支持配置有向/无向边、边标签、线型样式，用于描述上下游依赖、网络路径或逻辑关联
*   **节点级数据绑定**：单值节点和图表节点可绑定数据源，让拓扑图具备动态状态表达能力
*   **定时自动刷新**：支持配置刷新频率（如 30 秒、1 分钟），让关键状态持续更新
*   **编辑辅助工具**：
    *   **缩放与适配画布**：支持放大缩小、一键适配可视区域
    *   **撤销重做**：编辑操作可回溯，降低误操作成本
    *   **迷你图浏览**：画布缩略图辅助定位，便于在大图中快速导航
    *   **右键菜单**：节点与画布右键菜单提供快捷操作

### 1.4 架构图

#### 核心定位

架构图是面向**资源结构表达**的专用载体，适合用于系统架构展示、资源盘点、方案评审和变更前后的结构对比。相比拓扑图关注动态关系，架构图更关注静态资源结构的清晰表达。

#### 核心能力

*   **多资源图标体系**：
    *   **平台资源图标**：BlueKing Lite 内置平台组件图标
    *   **CMDB 图标库**：与配置管理联动的资源图标
    *   **云资源图标**：支持 AWS、阿里云、腾讯云等多类云资源图标
*   **统一画布编排**：在同一画布中组织视图、图标、颜色和结构层次，支持分层展示不同类型资源
*   **结构克隆与复用**：架构图配置可保存为模板，类似场景可快速复制改编
*   **跨角色沟通友好**：图形化表达降低理解门槛，适合技术评审、管理汇报等需要多方协作的场景
*   **版本化维护**：架构图作为长期资产可持续维护更新，避免每次评审都从零绘制

## 2. 设置中心（Settings）

设置中心用于管理运营分析的底层接入能力，决定页面能连什么数据、如何连以及谁可以用。它不是附属配置页，而是整个分析工作台可用性和复用性的基础。

### 2.1 数据源管理

#### 核心定位

数据源管理解决的是「数据从哪里来、如何被复用、适合以什么方式展示」的问题。它将取数路径、参数模板、标签归类、图表类型和归属团队统一纳入一个配置入口，是实现标准化分析资产的前提。

#### 核心能力

*   **REST API 统一接入**：通过定义 REST API 路径，让 CMDB、监控、告警、日志等内部能力以一致方式接入
*   **多命名空间关联**：单个数据源可关联多个命名空间，实现跨环境、跨域取数后统一汇总展示
*   **参数模板配置**：支持五类参数类型：
    *   **字符串**：文本型过滤条件
    *   **数字**：数值型参数
    *   **布尔**：开关型参数
    *   **日期**：日期选择器
    *   **时间范围**：时间段选择器，常用于全局时间驱动
*   **参数值类型**：支持**固定值**（不可编辑）与**可编辑值**（用户可调整）两种模式
*   **图表类型约束**：可为数据源限定可用图表类型（`line`、`bar`、`pie`、`single`），帮助页面设计者快速选择合适展示方式
*   **标签化治理**：支持关联多个数据源标签（如 CMDB、监控、告警、日志、业务、其他），便于后续检索、筛选和权限控制
*   **团队归属控制**：可为数据源配置归属团队，实现组织维度的数据可见性隔离
*   **启用状态管理**：支持启用/停用切换，停用后不可在新建组件时选用，但不影响已配置的历史组件

### 2.2 命名空间管理

#### 核心定位

命名空间管理用于维护外部数据环境的连接上下文，解决的是「平台通过什么连接方式访问外部数据能力」的问题。每个命名空间代表一个独立的数据环境，如不同集群、不同区域或不同业务线的数据入口。

#### 核心能力

*   **连接信息统一维护**：集中管理命名空间名称、NATS 命名空间（消息主题前缀）、账号、密码和域名信息
*   **TLS 安全连接**：支持在命名空间级别启用 TLS（`tls://` 协议），满足企业级网络安全要求
*   **密码加密存储**：命名空间密码在保存时自动进行 AES 加密，数据库中不存储明文，降低泄露风险
*   **动态协议适配**：系统根据 `enable_tls` 字段自动选择 `nats://` 或 `tls://` 协议构建连接 URL
*   **连接容错机制**：单个命名空间连接异常不阻断其他命名空间数据返回，保证分析页面可用性
*   **生命周期管理**：支持新增、编辑、删除和启用/停用状态管理，便于持续维护数据访问链路
*   **多数据源复用**：命名空间配置完成后，可被多个数据源重复引用，减少重复配置

> **界面指引：**
>
> 当前建议补充「命名空间列表与编辑抽屉」截图，展示名称、NATS 命名空间、账号、密码、域名、TLS 开关等配置项。
>
> *   **图表解读**：命名空间是数据接入的安全网关。TLS 支持与密码加密机制确保即使在外网环境也能安全访问内部数据能力。
