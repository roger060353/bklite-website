# 功能介绍

## 1. 分析视图（View）

分析视图是运营分析的主工作台，负责承载目录管理、分析对象切换和具体内容展示。相比单页报表工具，BK-Lite 运营分析更强调“资产化管理”，让团队能够围绕业务场景持续沉淀仪表盘、拓扑图和架构图。

### 1.1 目录与分析资产管理

#### 核心定位

目录用于组织分析资产，是运营分析能够长期沉淀和协同复用的前提。平台通过目录树把目录、仪表盘、拓扑图和架构图统一组织到同一结构中，便于团队按业务域、专题或职责范围管理内容。

#### 核心能力

*   **统一树形组织**：支持在同一目录树中统一管理目录、仪表盘、拓扑图和架构图，避免分析内容散落在多个孤立页面中。
*   **分层沉淀分析资产**：支持多层级目录组织方式，适合按业务系统、专题治理和团队职责沉淀长期分析成果。
*   **就地创建与维护**：用户可在目录树中直接新增、编辑、删除目录和分析对象，减少在不同管理页面之间切换的成本。
*   **删除保护与结构约束**：当目录下仍存在子对象时，不鼓励直接删除，帮助团队避免误操作破坏已沉淀的分析结构。
*   **未保存离开提醒**：当用户切换页面或对象时，平台会对未保存变更进行提示，减少编辑过程中内容丢失的风险。

>
> 当前建议补充「目录树与内容区」截图，用于展示目录、仪表盘、拓扑图和架构图在同一工作台中的组织方式。
>
> *   **图表解读/配置逻辑**：目录树不是简单导航栏，而是分析资产的管理入口。它决定团队如何沉淀内容、如何共享页面，以及如何把临时分析变成长期可复用资产。

### 1.2 仪表盘

#### 核心定位

仪表盘适合承载指标总览、趋势对比和专题观察，是运营分析中最适合做日常巡检和运营总览的展示载体。

#### 核心能力

*   **网格化自由布局**：支持通过网格布局组织多个组件，适合将趋势、对比、占比和单值信息集中到同一页面中展示。
*   **多图表类型支持**：支持折线图、柱状图、饼图和单值图，能够适配趋势观察、分类对比和关键状态展示等常见场景。
*   **统一时间范围控制**：当页面包含时间型参数时，可通过全局时间选择器统一驱动多个组件，便于跨指标横向对比。
*   **按数据源模板配置参数**：仪表盘组件可复用数据源中定义的参数模板，降低重复配置成本，提升页面一致性。
*   **保存与刷新机制**：支持页面保存和主动刷新，适合在值班巡检和专题回看过程中持续使用。

### 1.3 拓扑图

#### 核心定位

拓扑图用于表达对象关系、依赖链路和节点状态，适合处理单一指标难以说明全貌的问题定位场景。

#### 核心能力

*   **多类型节点编排**：支持图标节点、基础图形、文本节点、单值节点和图表节点，便于在同一画布中同时表达结构与状态。
*   **连线与标签表达**：支持配置边、标签与样式，用于描述上下游依赖关系、网络路径或逻辑关联。
*   **节点级数据绑定**：单值节点和图表节点可绑定数据源，让拓扑不只是静态结构图，而是能表达关键状态变化的分析画布。
*   **定时刷新能力**：对需要持续观察的节点支持定时刷新，适合承载运行态观察场景。
*   **编辑辅助能力**：支持缩放、适配画布、撤销重做和迷你图浏览，降低复杂画布的编辑难度。

> **界面指引：**
>
> 当前建议补充「拓扑图编辑页」截图，用于展示节点工具栏、画布区域、连线关系和属性编辑面板。
>
> *   **图表解读/配置逻辑**：拓扑图的价值不在于“画一张图”，而在于把关系结构和关键状态放在同一张画布里，让处理人能同时看到问题在哪里、与谁相关、影响到什么。

### 1.4 架构图

#### 核心定位

架构图用于表达资源结构、系统分层和平台拓扑，适合用于架构盘点、设计评审、变更沟通和跨角色协作场景。

#### 核心能力

*   **多资源图标体系**：支持平台资源、CMDB 图标以及多类云资源图标，适合组织复杂系统的结构表达。
*   **统一画布编排**：支持在同一画布中组织视图、图标、颜色和结构层次，帮助团队用一致方式表达架构关系。
*   **适合跨角色沟通**：相比表格或文字说明，架构图更适合在评审、汇报和复盘时快速建立共同理解。
*   **保存与持续维护**：架构图可以作为长期资产持续维护，不必每次都从零绘制，适合沉淀团队的系统认知。

## 2. 设置中心（Settings）

设置中心用于管理运营分析的底层接入能力，决定页面能连什么数据、如何连以及谁可以用。它不是附属配置页，而是整个分析工作台可用性和复用性的基础。

### 2.1 数据源管理

#### 核心定位

数据源管理解决的是“数据从哪里来、如何被复用、适合以什么方式展示”的问题。它将取数路径、参数模板、标签归类、图表类型和归属团队统一纳入一个配置入口。

#### 核心能力

*   **统一数据源定义**：通过 REST API 路径定义取数入口，让不同外部能力能够以一致方式进入运营分析。
*   **多命名空间关联**：单个数据源可关联多个命名空间，适合跨环境或跨域取数后统一汇总展示。
*   **参数模板配置**：支持字符串、数字、布尔、日期和时间范围等参数类型，并区分固定值、过滤项和动态请求参数。
*   **图表类型约束**：可为数据源限定可用图表类型，帮助页面设计者更快选择合适的展示方式。
*   **标签化治理**：支持通过标签管理数据源分类，便于后续检索、治理和复用。
*   **团队归属控制**：可为数据源配置归属团队，避免底层能力无边界扩散。

> **界面指引：**
>
> 当前建议补充「数据源列表与编辑抽屉」截图，用于展示名称、REST API、命名空间、标签、图表类型和参数模板配置区域。
>
> *   **图表解读/配置逻辑**：数据源页的本质是把零散接口能力沉淀成标准化分析资产。只有这里配置清楚，后续仪表盘和画布的复用成本才会足够低。

### 2.2 命名空间管理

#### 核心定位

命名空间管理用于维护外部连接上下文，解决的是“平台通过什么连接方式访问外部数据能力”的问题。

#### 核心能力

*   **连接信息统一维护**：集中管理命名空间名称、NATS 命名空间、账号、密码和域名信息，避免分散在多个页面重复维护。
*   **TLS 安全连接支持**：支持在命名空间级别启用 TLS，适合对接更严格的企业环境。
*   **密码加密存储**：命名空间密码在保存时会加密处理，降低明文保存带来的安全风险。
*   **启停与生命周期管理**：支持新增、编辑、删除和状态管理，便于持续维护数据访问链路。
*   **供数据源复用**：命名空间配置完成后，可被多个数据源重复引用，减少接入维护成本。

## 3. 权限与协作边界

权限能力贯穿视图和设置两个域，决定哪些人可以查看内容、谁可以创建资产、谁可以维护底层接入。对于需要在多个团队之间共享分析结果的场景，这是运营分析能够真正落地的关键保障。

### 3.1 核心定位

权限控制并不是为了增加操作门槛，而是为了确保分析资产和底层接入能力在共享时仍然可控、有边界、可审计。

### 3.2 核心能力

*   **查看与维护权限分离**：视图域和设置域可分别控制查看、新增、编辑、删除等动作，降低误改风险。
*   **目录与图表分层授权**：目录维护权限与分析对象维护权限分开配置，适合不同角色协同使用。
*   **设置域独立授权**：数据源和命名空间分别具备独立的 View、Add、Edit、Delete 权限，便于平台管理员控制底层接入能力。
*   **组织维度可见性**：数据源和分析对象可结合团队归属进行控制，更符合企业内部协作方式。
