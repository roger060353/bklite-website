# 产品介绍

## 一、产品简介

**BlueKing Lite 运营分析**是面向轻量级运维场景打造的统一数据可视化与运营决策工作台。它将分散在 CMDB、监控、告警、日志及业务系统中的多源数据整合到同一分析空间，通过仪表盘、拓扑图、架构图三种分析载体，帮助团队实现从「看见数据」到「理解关系、发现风险、支撑决策」的完整分析闭环。基于命名空间机制，运营分析还可以连接不同区域的 NATS 数据入口，在多地域 BlueKing Lite 通过 NATS 组网后，由主区域统一汇聚和展示各边缘区域数据，形成“边缘自治、中心可见”的分析能力。

与传统报表工具不同，BlueKing Lite 运营分析更关注**分析资产的长期沉淀与团队协同复用**：

*   **目录化管理**：支持以树形目录组织仪表盘、拓扑图和架构图，让分析成果按业务域、专题和场景有序沉淀
*   **多源数据融合**：单个数据源可关联多个命名空间，平台自动并行拉取并聚合返回，即使个别数据源异常也不影响整体分析
*   **边缘自治组网**：可通过命名空间连接不同区域的 NATS 服务，在各区域 BlueKing Lite 保持本地自治的同时，由主区域统一获取并展示跨区域数据
*   **安全可控共享**：结合细粒度权限控制与组织维度数据隔离，让分析资产既能跨团队共享，也能在安全边界内可控访问


## 二、核心优势

*   **统一数据分析入口**：通过数据源与命名空间管理能力，把 CMDB、监控、告警、日志及业务接口统一纳入同一工作台，告别多系统来回切换的信息割裂
*   **三种分析载体协同表达**：同一分析空间内同时支持**仪表盘**（指标趋势）、**拓扑图**（关系定位）、**架构图**（资源结构），覆盖从观察到决策的完整分析链路
*   **目录化资产沉淀**：支持多层级目录组织分析资产，让临时报表进化为可长期复用的团队知识库，避免分析成果散落在个人页面中
*   **智能数据聚合机制**：多命名空间并行取数，单点异常自动降级，请求自动携带用户与团队上下文，保障数据权限语义一致性
*   **边缘自治与中心汇聚兼得**：通过命名空间绑定不同区域 NATS，主区域可在不破坏各边缘区域独立运行的前提下统一获取和展示多地域数据
*   **企业级安全接入**：命名空间支持 TLS 加密连接、密码 AES 加密存储，满足企业级数据安全与合规要求
*   **开箱即用的内置能力**：系统预置 CMDB、告警、监控、日志、业务等数据源标签和常用数据模板，降低初次使用门槛

## 三、核心能力全景

### 1. 目录化的分析工作台

采用左侧目录树 + 右侧内容区的经典双栏布局，统一管理**目录、仪表盘、拓扑图、架构图**四类分析对象。支持按业务系统、运维专题或治理目标建立最多 **3 层**目录结构，让分析内容具备清晰的归属关系，便于团队协同与资产复用。

### 2. 面向指标观察的仪表盘

基于网格自由布局组合多种可视化组件，支持**折线图、柱状图、饼图、单值图**等图表类型。通过全局时间选择器和公共过滤条件，可统一驱动多个组件联动刷新，适合承载日常巡检、专题运营和趋势对比等场景。

### 3. 面向关系定位的拓扑图

提供可视化画布用于表达对象关系、依赖链路和节点状态。支持**图标节点、基础图形、文本节点、单值节点、图表节点**等多种节点类型，可配置连线、标签和样式，并支持节点级数据绑定与定时刷新，让拓扑图从静态结构图升级为动态状态观测画布。

### 4. 面向资源表达的架构图

专为系统架构展示、资源盘点和变更沟通设计。支持平台资源、CMDB 图标及多类云资源图标，可在同一画布中组织视图、颜色和结构层次，帮助技术与非技术角色快速建立共同理解。

### 5. 灵活的数据源与命名空间管理

*   **数据源管理**：定义 REST API 取数路径、参数模板（支持字符串、数字、布尔、日期、时间范围类型）、可用图表类型和数据源标签
*   **命名空间管理**：维护连接地址、账号、TLS 开关等连接信息，支持密码加密存储与生命周期管理
*   **一对多关联**：单个数据源可关联多个命名空间，实现跨环境、跨域数据统一分析

> **注意：**
> 运营分析更适合承担「统一观察、关系表达、专题复盘与决策支撑」的角色。对于超大规模实时计算或复杂算法建模场景，建议结合专门的数据平台协同使用。

## 四、典型应用场景

### 1. 运营全景总览

将资源变化、告警趋势、服务状态和业务关键指标整合进统一工作台。通过仪表盘集中呈现关键指标，通过拓扑图展示对象关系，帮助值班人员与管理层在同一视图下建立一致认知，快速把握系统整体健康度。

### 2. 故障影响范围定位

当核心服务异常时，单一指标往往难以说明影响范围。通过拓扑图组合单值节点与图表节点，处理人可以直观看到异常节点、上下游依赖和关联状态变化，辅助判断故障是局部波动还是链路级影响，缩短 MTTR。

### 3. 架构盘点与变更沟通

将分散在文档、表格和脑图中的架构信息统一沉淀为可维护的架构图。便于技术团队复盘资源关系，也便于向非研发角色汇报系统结构与变更影响边界，提升跨团队协作效率。

### 4. 团队分析资产沉淀

通过目录化组织和权限控制，把例行巡检看板、专题分析页、架构对照图逐步沉淀为团队公共资产，形成可继承、可复用、可持续优化的运维分析体系，避免人员变动导致的知识流失。

| **分析表达** | 单一图表类型 | 仪表盘+拓扑图+架构图，覆盖多元场景 |
| **资产管理** | 个人页面散落 | 目录化组织，支持 3 层层级沉淀 |
| **协作共享** | 权限控制粗放 | 细粒度操作权限 + 组织维度数据隔离 |
| **安全接入** | 基础连接能力 | TLS 加密 + 密码加密存储 + 启停管理 |

对于希望快速建立统一运营分析入口、或已有多个数据系统但缺少统一表达方式的团队，BlueKing Lite 运营分析提供了一套**开箱即用、可持续演进**的轻量级解决方案。它不是一页孤立的看板，而是能够陪伴团队持续成长的运营分析工作台。
