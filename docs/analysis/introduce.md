# 产品介绍

## 一、产品简介

BK-Lite 运营分析是面向轻量级运维场景打造的统一分析工作台，用于把分散在 CMDB、监控、告警、日志及业务系统中的数据组织到同一个分析空间中，帮助团队从“看见数据”进一步走向“理解关系、发现风险、支撑决策”。它不是单一的图表页面，而是一套围绕目录管理、数据接入、分析表达和团队协作构建的运营分析能力。

与只提供单一看板展示的可视化工具不同，BK-Lite 运营分析更关注运维分析资产的长期沉淀与复用。一方面，平台支持以目录树方式管理仪表盘、拓扑图和架构图，让团队可以按业务域、专题和场景持续沉淀分析成果；另一方面，平台提供数据源、命名空间和权限控制能力，让分析结果既能被复用，也能在合适的边界内安全共享。

对于一线运维人员，它提供统一的数据观察入口，减少在多个系统之间来回切换的成本；对于团队负责人，它提供更适合复盘、对比和洞察的分析表达方式，便于持续识别风险与优化空间；对于平台管理员，它提供命名空间接入、数据源配置、标签分类和权限控制能力，便于建立稳定、可持续演进的分析底座。

## 二、核心优势

*   **统一的数据分析入口**：通过数据源与命名空间管理能力，把 CMDB、监控、告警、日志及业务侧接口统一纳入同一分析工作台，减少跨系统切换带来的信息割裂。
*   **三种分析载体协同表达**：同一套分析空间内同时支持仪表盘、拓扑图和架构图，既能展示指标趋势，也能表达对象关系与资源结构，适应从观察到定位再到复盘的不同场景。
*   **目录化沉淀分析资产**：平台支持目录、仪表盘、拓扑图、架构图统一纳管，便于团队按业务域和专题组织内容，让一次构建的分析成果可以长期复用，而不是停留在临时报表层面。
*   **灵活的数据接入与汇总返回**：单个数据源可关联多个命名空间，平台会按配置向不同命名空间取数并汇总结果返回，即使个别命名空间异常，也不会阻断整体分析页面的展示。
*   **团队协作与权限边界清晰**：围绕查看、新增、编辑、删除等动作提供细粒度权限控制，并结合组织维度的数据隔离能力，让分析资产既可共享，也可控、可管、可审计。
*   **安全可控的接入机制**：命名空间支持 TLS 连接、密码加密存储和启停管理，适合把分析能力接入到企业现有数据环境中，而不是额外引入一套难以治理的可视化孤岛。

## 三、核心能力全景

### 1. 目录化的分析工作台

BK-Lite 运营分析采用左侧目录树加右侧内容区的工作台设计，支持围绕目录统一管理仪表盘、拓扑图和架构图。团队可以按业务系统、运维专题或治理目标建立多层级分析空间，让分析内容具备清晰的归属关系，而不是散落在多个个人页面中。对于需要长期维护的运营看板、专题复盘视图和架构观察页面，这种目录化管理方式更适合持续沉淀。

### 2. 面向指标观察的仪表盘

在仪表盘场景中，平台支持基于网格布局组合多种可视化组件，适合承载日常巡检、专题运营和趋势对比等分析需求。用户可结合公共过滤条件和时间范围统一观察不同数据源结果，并按需要选择折线图、柱状图、饼图和单值图等图表类型，将零散指标组织成更可读的运营视图。

### 3. 面向关系定位的拓扑图

当问题不仅体现在数值变化，还需要理解对象之间的连接关系时，拓扑图可以提供更直观的分析方式。平台支持在画布中组合图标节点、基础图形、文本节点、单值节点和图表节点，并配置连线、标签和样式，用于表达链路关系、依赖结构和关键状态。对于需要观察服务上下游、网络关系或告警影响面的场景，拓扑图比单纯的数字看板更容易帮助处理人建立问题上下文。

### 4. 面向资源表达的架构图

运营分析不仅用于看数据，也适合展示平台和业务的资源结构。架构图能力支持在同一画布中组织多种云资源与平台资源图标，并可结合 CMDB 图标库与多类资源包进行编排，适合用于系统架构展示、资源盘点、方案评审和变更前后的结构对比。对于需要同时面对技术团队与管理视角的场景，这种图形化表达更有利于统一理解。

### 5. 面向复用的数据源与命名空间管理

为了让分析能力真正落地，平台提供了独立的数据源与命名空间管理能力。管理员可以配置 REST API 数据源、参数模板、可用图表类型以及数据源标签，并将数据源关联到多个命名空间。分析页面在查询时会自动带上当前用户与团队上下文，使数据接入更贴近企业内部的权限语义，也便于在不同团队之间建立统一而可控的分析标准。

> **注意 / 安全最佳实践：**
> 运营分析更适合承担“统一观察、关系表达、专题复盘与决策支撑”的角色。对于超大规模实时计算或复杂算法建模场景，应结合专门的数据平台或算法平台协同使用，以获得更匹配的处理能力。

## 四、典型应用场景

### 1. 运营全景总览

当团队需要同时关注资源变化、告警趋势、服务状态和业务关键指标时，运营分析可将多来源数据整合进统一工作台，通过仪表盘集中呈现关键指标，通过拓扑图展示对象关系，帮助值班与管理角色在同一视图下建立一致认知。

### 2. 故障影响范围定位

当某项核心服务出现异常时，单条指标或单条告警往往难以快速说明影响范围。通过拓扑图与单值节点、图表节点的组合，处理人可以更直观看到异常节点、上下游依赖和关联状态变化，辅助判断故障是局部波动还是链路级影响。

### 3. 架构盘点与变更沟通

对于需要做系统盘点、上云规划、架构评审或重要变更沟通的团队，架构图可以把原本分散在文档、表格和脑图中的结构信息统一沉淀下来。这样既便于技术团队复盘资源关系，也方便非研发角色理解关键系统结构与影响边界。

### 4. 团队分析资产沉淀

在持续运营过程中，真正有价值的不是某一张临时图表，而是可以复用的分析方法。通过目录化组织和权限控制，团队可以把例行巡检看板、专题分析页、架构对照图逐步沉淀为公共资产，形成可继承、可复用、可持续优化的运维分析体系。

## 五、为什么选择 BK-Lite 运营分析

对于轻量级运维平台来说，分析能力的价值不只是“把数据画出来”，而是让团队能够在同一空间里持续组织数据、表达关系、沉淀认知并支撑决策。BK-Lite 运营分析的优势在于，它没有把分析能力割裂成彼此独立的报表工具、画布工具和接入工具，而是把目录管理、数据接入、权限控制和多种分析载体整合成一套统一能力。

它适合希望快速建立统一运营分析入口的团队，也适合已经拥有多个数据系统、但缺少统一表达与沉淀方式的组织。对于需要兼顾建设成本、协作效率和治理可持续性的企业来说，这不是一页孤立的看板，而是一套能够陪伴团队持续演进的运营分析工作台。
