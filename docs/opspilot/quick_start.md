# 快速入门

本文将指导您从零开始，在 OpsPilot 中配置模型、注入企业本地知识，并快速发布一个能够在企业微信或 Web 端对话交互的智能应用。

## 前置条件
*   已获取 OpsPilot 平台访问权限及所在所在团队的编辑权限。
*   已提前向云厂商开通对应大语言模型的 API 密钥（如 DeepSeek、通义千帆等）。

## 分步操作指引

### 1. 配置模型接入 (Provider)
有了计算“大脑”，智能体才能进行推演。
1.  导航至「模型 (Provider)」模块，点击「新建模型」。
2.  选择需要填写的模型大类（LLM、Embed 等）与厂商，输入您的 API 密钥等鉴权信息。
3.  点击「保存」并启用，将模型状态设置为可见。为了方便后续管理，您可以将同类模型划分至特定的业务分组中。

### 2. 注入企业私域知识 (Knowledge)
为模型赋予企业级“记忆”，避免“回答幻觉”。
1.  进入「知识库 (Knowledge)」模块，新建知识库并选择对应的基础 Embed 知识片段化模型。
2.  在知识库详情内上传本地核心文件。
3.  等待后台完成对文档的自动解析、预处理与分块。当显示“就绪”状态后，您的智能体就可以在检索时随时引用这部分内容了。

### 3. 创建智能体 (Skill)
组装模型和知识库，定义实际“干活”的 AI 执行角色。
1.  前往「智能体 (Skill)」模块，点击「新建」创建一个新的专属智能体。
2.  为该智能体编排「系统提示词」以设定其长期记忆与人设规则。
3.  绑定在步骤 1 配置的 LLM 模型，并开启 **RAG** 能力以关联步骤 2 建立的私有知识库。
4.  保存配置，并可在页面右侧的「测试窗」中发几句问询，检查其是否掌握了您的内部知识。

### 4. 编排工作流并发布应用 (Studio)
利用 ChatFlow ，将智能体组装成可对外暴露的服务形式。
1.  前往「工作台 (Studio)」模块，选择或者新建一个 **ChatFlow** 类型的应用实体。
2.  在可视化画布中，从左侧拖拽您的智能体与判路节点，连线组合成一条顺畅的自动处理流程。
3.  点击「渠道」卡片，选择需要的渠道类型（如 **企业微信机器人** 或 **钉钉**），将 webhook 地址或参数填入。
4.  将应用状态切换并确认「上线」。

## 结果验证与闭环

完成上述配置后，您的企业智能助手已正式起航：
1.  前往对应的您刚才配置的渠道（例如：在您的企业微信讨论群内），直接@该机器人进行对话提问，看它是否作出了预期的答复。
2.  您可随时返回「工作台 (Studio)」中的「应用日志」和「应用统计」功能：
    *   **业务审计闭环**：通过查看执行日志梳理模型处理耗时，确保问答行为符合企业预期。
    *   **使用量跟踪闭环**：结合趋势面板洞察哪些问题的请求频次最高，以推动针对该高频问题进行知识库重点更新。
