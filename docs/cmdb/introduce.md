# 产品介绍

## 一、简介

BlueKing Lite CMDB 是面向云原生时代打造的**智能化配置管理数据库**，以「图数据库驱动的关系建模」为核心架构，帮助企业构建从基础设施到应用服务的全栈资产数字孪生。不同于传统 CMDB 的"表格台账"模式，BlueKing Lite CMDB 将资产视为**活的数据实体**，通过自动发现持续感知环境变化，通过拓扑关系可视化呈现系统架构，通过智能订阅主动推送关键变更——让资产管理从"静态记录"升级为"动态治理"。

### 价值主张

| 角色 | 核心痛点 | BlueKing Lite CMDB |
|------|----------|---------------------|
| **资产管理员** | 资产口径不统一、模型标准难落地 | 可视化建模工具 + 模型复制能力，快速建立企业级资产标准 |
| **运维工程师** | 故障定位慢、影响范围难评估 | 拓扑关系可视化 + 变更追溯，分钟级定位问题根因 |
| **平台架构师** | 混合云资产分散、数据不一致 | 20+ 采集插件 + 自动发现，统一纳管 K8s/云/虚拟化/网络设备 |
| **安全合规员** | 资产变更无留痕、审计困难 | 全生命周期变更记录 + 订阅通知，满足合规审计要求 |

## 二、核心优势

### 图数据库原生架构

BlueKing Lite CMDB 底层采用 **FalkorDB 图数据库**存储资产关系，相比传统关系型数据库：

*   **多跳关系高效查询**：设备-接口-连接-对端设备，4 层以上拓扑关联查询毫秒级响应
*   **动态关系建模**：无需预先定义外键，模型关系可随业务演进灵活调整
*   **参数化 CQL 查询引擎**：内置 CQL 注入防护与参数化查询，兼顾灵活性与安全性

### 智能发现与持续同步

平台内置 **20+ 专业采集插件**，覆盖容器、云原生、虚拟化、网络、数据库、中间件六大领域：

*   **K8s 全景发现**：自动识别集群→命名空间→工作负载→Pod→节点的 5 层拓扑关系
*   **网络设备智能识别**：基于 SOID（sysObjectID）特征库精准识别设备型号、品牌、类型
*   **关系自动恢复**：网络设备接口连接关系、虚拟机与宿主机关系自动推导补全

### 企业级安全与合规

*   **采集凭据加密存储**：任务凭据 AES 加密落库，接口返回统一脱敏
*   **组织级权限隔离**：模型、实例、采集任务按组织范围管控，多团队安全协作
*   **全生命周期审计**：实例创建、修改、删除、关联变更全程留痕，支持变更详情追溯

### 数据订阅与主动通知

突破传统 CMDB "被动查询"模式，支持**主动式数据订阅**：

*   **三类触发器**：属性变更、关联关系变更、到期提醒
*   **合并降噪机制**：同一实例多次属性变更合并为单条通知，避免消息轰炸

### 轻量落地与快速扩展

*   **模型复制加速**：一键复制模型属性、字段分组、关联关系配置，标准模型分钟级复用
*   **公共选项库**：状态、等级、分类等枚举值跨模型统一维护，变更自动同步
*   **多语言开箱即用**：内置中英文语言包

## 三、典型应用场景

### 场景一：混合云资产统一纳管

**挑战**：企业同时使用阿里云、腾讯云、私有云 K8s、VMware，资产分散在各平台控制台，缺乏统一视图。

**BlueKing Lite CMDB 方案**：
1.  配置云厂商采集任务，自动同步 ECS、VPC、RDS 等资源
2.  配置 K8s 集群接入，自动发现容器资源与节点关系
3.  配置 VMware 采集，同步虚拟化资产
4.  通过「资产视图」获得跨云资产总览，通过「全文检索」秒级定位任一实例

**收益**：资产盘点时间从天级缩短到分钟级，跨云依赖关系清晰可见。

### 场景二：故障快速定位与影响分析

**挑战**：核心服务告警，需要快速判断「哪些下游服务受影响」「近期是否有变更」。

**BlueKing Lite CMDB 方案**：
1.  在「全文检索」输入服务名，定位到目标实例
2.  进入「拓扑视图」，按层展开上下游依赖链路
3.  查看「变更记录」，确认近期是否有配置调整或自动发现写入
4.  如需持续关注，配置「数据订阅」，后续变更自动推送

**收益**：平均故障定位时间（MTTR）降低 60% 以上。

### 场景三：资产配置标准化治理

**挑战**：各业务线自建 Excel 台账，字段定义不一致，数据质量差。

**BlueKing Lite CMDB 方案**：
1.  在「模型管理」定义企业级资产标准（字段、枚举、约束）
2.  使用「模型复制」能力，快速为各业务线复用标准模型
3.  使用「公共选项库」统一状态、环境、责任人等枚举值
4.  通过「字段分组」优化展示结构，提升录入体验
5.  历史数据通过「批量导入」快速迁移

**收益**：资产数据一致性提升，跨部门协作效率显著改善。

### 场景四：安全合规与审计支撑

**挑战**：等保合规要求资产变更可追溯，但传统方式缺乏有效工具。

**BlueKing Lite CMDB 方案**：
1.  启用「变更记录」自动记录所有实例变更（操作人、时间、前后值）
2.  对敏感资产配置「数据订阅」，关键变更实时通知安全团队
3.  采集任务启用「审批后写入」，避免未授权数据进入台账
4.  定期导出变更记录，满足合规审计要求

**收益**：审计准备时间从数周缩短到数小时。

## 五、为什么选择 BlueKing Lite CMDB

### 与传统 CMDB 对比

| 维度 | 传统 CMDB | BlueKing Lite CMDB |
|------|-----------|--------------|
| **数据模型** | 关系型数据库，表结构僵化 | 图数据库，关系即数据，灵活扩展 |
| **数据时效** | 依赖人工维护，数据快速过时 | 自动发现持续同步，数据鲜活可信 |
| **关系表达** | 外键关联，查询深度受限 | 图遍历查询，多跳关系高效分析 |
| **使用模式** | 被动查询，人找数据 | 主动订阅，数据找人 |
| **落地成本** | 实施周期长，定制开发多 | 开箱即用，轻量起步 |

| 维度 | 开源 CMDB | BlueKing Lite CMDB |

| **拓扑展示** | 基础展示，交互有限 | 分层展开、节点扩展、影响分析 |
| **多租户** | 自行实现 | 组织级隔离内置支持 |
| **数据订阅** | 无或需额外开发 | 内置三类触发器，事件驱动 |
