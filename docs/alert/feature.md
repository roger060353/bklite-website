# 功能介绍

## 1. 告警（Alert）

告警是原始 Event 经相关性规则聚合后形成的处理对象。相比单条事件，告警更强调责任、上下文与处置动作，是值班人员日常工作的核心入口。

### 1.1 核心定位

告警用于承接需要被处理的问题单元。它既保留原始事件的上下文信息，也提供统一的状态流转和责任归属能力，帮助团队把“看到异常”升级为“开始处理问题”。

### 1.2 核心能力

*   **多维筛选查看**：支持按告警级别、状态、来源、时间范围、是否为我的告警等维度快速筛选，帮助值班人员优先处理最关键的问题。
*   **关联事件回溯**：每条告警都可以查看其关联事件，用于理解告警是如何形成的，并辅助判断聚合规则是否符合预期。
*   **标准状态流转**：支持分派、认领、转派、关闭等标准操作，让处理过程具备明确的责任归属和可追踪性。
*   **自动恢复与自动关闭**：当外部系统上送恢复事件时，平台可自动判断告警恢复；对长期未处理且满足条件的告警，平台也可按规则自动关闭。
*   **通知状态可感知**：在处置过程中，平台会记录通知发送结果与状态，帮助团队了解告警是否已被有效触达。

## 2. 事故（Incident）

Incident 用于承接已经具备更高业务影响的问题。它不是对告警的简单重命名，而是将需要团队协同处理的异常纳入更高等级的管理对象。

### 2.1 核心定位

当一条或多条告警已经指向同一高影响问题时，可以将其升级为 Incident，以统一跟踪处置进展、组织处理人员并集中查看关联信息。

### 2.2 核心能力

*   **告警升级事故**：支持从告警列表中选择对象升级为事故，适用于业务中断、核心链路异常等高影响场景。
*   **多告警统一承接**：事故对象可关联多条告警，便于在一个工作对象中统一管理上下文与进展。
*   **事故状态管理**：支持事故的认领、关闭与重开，帮助团队围绕事故对象推进响应与收敛。
*   **协同信息集中查看**：在事故详情中可查看基础信息、关联告警及处理过程，便于多角色共享同一问题视图。

> **界面指引：**
>
> ![事故列表](https://static.cwoa.net/13c556375fb440baa763059fd6df2512.png)
>
> *   **图表解读/配置逻辑**：事故页面适合处理中高影响的问题。相比告警列表，它更强调协同视角，帮助用户从“单条异常处理”切换为“同一问题统一推进”。

## 3. 集成中心（Integration）

集成中心用于管理事件接入来源，是告警中心的入口层能力。平台通过告警源统一管理不同系统的接入方式、鉴权信息与使用状态。

### 3.1 核心定位

集成中心解决的是“事件从哪里来、如何安全地进入平台、如何验证接入是否成功”的问题。它将接入配置、指南查看和事件核验集中在同一入口中，便于平台管理员维护标准接入方式。

### 3.2 核心能力

*   **多接入方式支持**：支持 REST API、NATS 等多种接入方式，适应不同系统的集成需求。
*   **源级鉴权管理**：每个告警源都具备独立的接入配置与密钥，便于按来源管理安全边界。
*   **接入指南与事件查看**：可在告警源详情中查看接入说明、鉴权参数和最近事件情况，便于快速定位接入问题。
*   **告警源生命周期管理**：支持新增、编辑、停用与删除告警源，帮助平台持续维护统一接入体系。

> **界面指引：**
>
> ![告警源](https://static.cwoa.net/d77279aa03604616947a03c7a395f191.png)
>
> *   **图表解读/配置逻辑**：集成中心既是接入配置入口，也是故障排查入口。若接入后没有看到预期告警，通常应先在这里确认源配置、鉴权参数和事件接收情况，再继续排查规则与处置链路。

## 4. 设置中心（Settings）

设置中心负责沉淀告警治理规则，让平台从“被动收消息”升级为“主动管理问题”。这里包括事件侧相关性规则、分派策略、屏蔽策略、系统设置和操作日志等核心能力。

### 4.1 相关性规则

相关性规则负责定义 Event 如何聚合为 Alert，是告警中心降噪与提炼价值的关键能力。

#### 核心能力

*   **规则化聚合**：通过匹配条件与分组字段定义事件如何聚合，减少重复告警。
*   **多窗口策略**：支持固定窗口、滑动窗口与会话窗口，以适应不同异常场景。
*   **关闭时长配置**：支持为规则生成的告警设置自动关闭时间，控制长期悬挂问题。

### 4.2 告警分派

分派策略负责将符合条件的告警自动分配到责任人或责任团队，提升问题进入处理流程的效率。

#### 核心能力

*   **条件匹配分派**：按告警字段与规则匹配责任人或责任团队，减少人工转派。
*   **生效时间控制**：支持单次、每日、每周、每月等时间段配置，适应不同值班安排。
*   **通知联动**：告警分派后可触发通知渠道，将问题同步给处理人。
*   **提醒频率控制**：可根据告警等级配置未响应情况下的提醒频率，避免责任悬空。

> **界面指引：**
>
> ![告警分派](https://static.cwoa.net/65bec31304774ff8b27a47dc853af3aa.png)
>
> *   **图表解读/配置逻辑**：分派策略的重点不是“填谁来处理”，而是定义“什么类型的问题在什么时间自动交给谁”。合理的分派策略可以显著减少人工判断与转派开销。

### 4.3 屏蔽策略

屏蔽策略用于过滤低价值、已知或维护窗口内不需要进入告警处置流程的事件，进一步降低噪声。

#### 核心能力

*   **按条件屏蔽事件**：支持按来源、资源、标题或其他字段条件屏蔽不需要处理的事件。
*   **按时间段生效**：支持单次、每日、每周、每月等时间范围配置，适用于维护窗口或周期性操作场景。
*   **前置降噪**：命中屏蔽策略的事件不会进入后续告警处理链路，减少无效告警生成。

> **界面指引：**
>
> ![屏蔽策略](https://static.cwoa.net/89814c631fc441b3915f72f9f5caa393.png)
>
> *   **图表解读/配置逻辑**：屏蔽策略适合治理“明知无须处理”的事件，例如计划内维护或重复性低价值通知。它的价值在于让团队把注意力留给真正需要响应的问题。

### 4.4 全局配置与操作日志

全局配置用于维护告警中心的系统级行为；操作日志则负责记录关键变更与处置动作，是平台治理和审计的重要组成部分。

#### 核心能力

*   **系统行为统一管理**：通过全局配置控制告警中心的部分平台级行为，保持处理口径一致。
*   **操作留痕审计**：记录策略变更、系统配置调整与告警处置动作，便于复盘和问责。
*   **治理效果回看**：结合操作记录、通知状态与趋势统计，帮助团队持续优化规则与流程。

> **界面指引：**
>

>
> *   **图表解读/配置逻辑**：全局配置体现的是平台处理策略的一致性，而操作日志体现的是治理动作的透明度。两者结合，可以让告警中心从“能用”进一步走向“可治理”。
