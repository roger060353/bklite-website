# 产品介绍

## 一、产品简介

BK-Lite 告警中心是面向轻量级运维场景打造的统一事件收敛与处置平台，用于承接来自监控系统、业务系统、流水线平台以及第三方工具的多源事件输入，并将分散、重复、上下文不足的原始事件转化为可分派、可协同、可审计的告警与事故对象。

与只负责“发通知”的传统告警能力不同，BK-Lite 告警中心更关注告警从产生到关闭的完整闭环。一方面，平台通过相关性规则、分组维度与时间窗口，将大量重复事件收敛为可处理的 Alert，降低值班人员面对噪声时的判断负担；另一方面，平台围绕分派、认领、转派、关闭、自动恢复、自动关闭和 Incident 升级等流程构建标准化处置路径，让告警真正进入协同处理而不是停留在消息提示层。

对于值班运维人员，它提供统一的告警视图、明确的责任归属和可追溯的处理过程；对于团队负责人，它提供分派策略、屏蔽策略、相关性规则和趋势统计能力，便于持续优化处置效率；对于平台管理员，它提供告警源管理、系统设置和权限控制能力，便于建立标准化、可持续演进的事件治理体系。

## 二、核心优势

*   **多源事件统一接入**：平台支持通过 REST API 与 NATS 双通道接入事件，并通过告警源适配能力对不同来源的数据做标准化处理，让监控、流水线、Webhook 与自定义系统事件进入同一治理入口。
*   **规则驱动的聚合降噪**：基于 `match_rules`、`group_by` 与时间窗口策略，平台能够将同一主题、同一对象或同一异常上下文中的多个 Event 自动聚合为一个 Alert，避免重复告警淹没真正需要处理的问题。
*   **从通知到处置的责任闭环**：平台不仅能触发通知，还能围绕分派、认领、转派、关闭等标准动作建立责任流转路径，并支持按个人、团队和生效时间配置自动分派策略，减少人工协调成本。
*   **自动恢复与自动关闭兜底**：当外部系统推送恢复事件后，平台可基于 `external_id` 自动关联历史事件并完成恢复判断；对于长期未处理但满足策略条件的告警，还可自动关闭，降低“僵尸告警”堆积。
*   **从告警升级到事故协同**：当单条或多条告警已具备业务影响时，可升级为 Incident 进行更高等级的协同处理，让重大问题从“个人处理”转向“团队响应”。
*   **处置过程可审计、可复盘**：告警操作、策略调整与系统设置变更均会形成操作日志，配合告警趋势与通知状态信息，帮助团队在事后复盘时还原处置过程、评估治理效果。

## 三、核心能力全景

### 1. 统一的事件治理入口

BK-Lite 告警中心以 Event、Alert、Incident 三层模型组织事件治理流程。原始事件首先进入平台并完成标准化，再通过规则引擎聚合为可处理告警；对于影响范围更大、处置协同要求更高的问题，则可以进一步升级为 Incident。这样的分层设计既避免了原始事件直接暴露给处理人带来的噪声，又让重大故障具备更清晰的升级路径。

### 2. 可配置的相关性规则引擎

平台提供事件侧相关性规则配置能力，支持按来源、字段、标签与关键属性进行匹配，并结合分组字段与窗口类型定义聚合边界。对于需要按连续异常判断的问题，可使用滑动窗口；对于需要基于事件间隔判断的问题，可使用会话窗口。规则不依赖代码改造即可调整，更适合持续演进的运维场景。

### 3. 面向处理流程的告警工作台

告警中心围绕实际处置流程设计页面与操作能力。处理人可以在统一列表中查看活跃告警、快速筛选自己负责的对象、查看关联事件和操作记录，并在合适时机执行认领、转派、关闭或升级事故等动作。对于仍未响应的告警，平台还可结合提醒任务与通知渠道进行周期性提醒，确保责任不悬空。

### 4. 可控的策略中心

在设置中心中，团队可以对自动分派、屏蔽规则、相关性规则和全局配置进行维护。这样的设计让治理逻辑从零散的脚本和人工约定中抽离出来，沉淀为可审计、可复用、可持续优化的策略体系。

## 四、典型应用场景

### 1. 基础设施异常的统一收敛

在主机、数据库、中间件等基础设施场景中，同一对象的异常往往会表现为多个指标同时波动。BK-Lite 告警中心可将这些具有相同资源上下文的事件聚合成单条告警，帮助运维人员更快识别“同一个问题”的真实影响，而不是被多条重复事件打断节奏。

### 2. 流水线与任务失败的持续跟踪

对于构建失败、发布异常、批处理任务中断等场景，平台可基于窗口规则判断异常是否持续存在，并将短时间内重复出现、但本质指向同一问题的事件收敛为统一告警，便于研发和运维团队持续跟踪而不是重复接收相似通知。

### 3. 重大问题的事故化协同

当告警已经涉及业务中断、核心链路异常或需要多角色协同处置时，值班人员可以将告警升级为 Incident，纳入更正式的事故处理流程。这样既保留了原始告警上下文，也为更高等级的应急响应提供了统一入口。

### 4. 合规审计与处置复盘

在重视运维审计与治理优化的团队中，平台提供的操作日志、状态流转与通知状态信息可以帮助管理者回答“谁处理了什么问题”“策略如何变更”“为什么没有及时响应”等关键问题，让告警治理具备可追踪、可复盘的管理价值。

## 五、为什么选择 BK-Lite 告警中心

对于轻量级运维平台而言，告警系统的价值不在于“能不能发出一条消息”，而在于能否把事件转化为真正可执行的处置流程。BK-Lite 告警中心的优势在于，它用相对轻量的建设方式，把事件接入、聚合降噪、责任分派、自动恢复、事故升级和审计留痕串成了一条完整链路。

它既适合希望快速建立统一告警入口的团队，也适合已经拥有多种监控与事件来源、但缺乏统一处置机制的组织。对于需要在成本、效率与治理能力之间取得平衡的企业来说，这不是一个只会“推送通知”的工具，而是一套能够持续提升运维响应质量的告警中心能力。
